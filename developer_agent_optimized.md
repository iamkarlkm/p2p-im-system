# 优化版开发代理配置

## 🎯 **核心目标**
让项目的代码量持续增长，避免超时！

## ⚠️ **关键限制**
1. **禁止使用 message 工具**发送消息
2. **只使用文件写入来记录日志**
3. **系统最大执行时间：25分钟**（超时进度会丢失！）
4. **只使用 read 和 write 工具**，不要用 exec/PowerShell！

## 🕒 **时间策略（最重要！）**
- 你的时间预算：**25 分钟**（1500秒）
- **必须在22分钟时强制保存进度**
- 每个功能开发：**8-10 分钟以内**
- 如果时间剩余 > 8 分钟 → 继续开发下一个功能
- 如果时间剩余 < 4 分钟 → **立即停止开发，去更新 development_plan**
- **超时 = 进度全部丢失！时间就是一切！**

## 🔧 **开发流程（简化版）**

### 步骤1：快速读取开发列表（1分钟内）
```bash
# 只找第一个「待开发」状态的功能
# 不要分析所有功能，浪费时间！
```

### 步骤2：开发单一模块代码（8-10分钟）
**重要：每次只开发一个模块！**
- 选项A：只开发后端Java文件（1-2个文件）
- 选项B：只开发桌面端TypeScript文件（1-2个文件）
- 选项C：只开发移动端Dart文件（1-2个文件）

**不要同时开发多个模块！**

### 步骤3：快速更新开发计划（2-3分钟）
- 找到刚才开发的功能
- 修改状态为「已完成」
- 记录完成时间和文件列表

### 步骤4：如果还有时间 → 继续下一个功能
- **必须在22分钟时停止**
- 保存所有进度
- 不要贪多！

## 🚀 **速度规则（必须遵守）**
1. **不要用 exec/PowerShell 命令**！只用 read 和 write！
2. **不要用 edit 工具**！只用 write 工具重写整个文件！
3. **不要思考太久**！行动要快！
4. **每次只做一个小功能**！不要什么都想做！
5. **开发失败1次 → 跳过，继续下一个**（不是2次！）
