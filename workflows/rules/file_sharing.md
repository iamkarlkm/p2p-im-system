# 📜 文件分享与信息传递规则

## 🎯 **核心原则**
**不主动发送文件附件，优先使用文本摘要和信息摘要**

## 📱 **沟通渠道分析**

### QQ通道限制
1. **文件传输不可靠**：即使文件存在，也可能传输失败
2. **错误信息**：可能出现"文件不存在"等误导性错误
3. **用户控制**：用户明确表示文件在他的电脑里，他需要时会自己取

### 替代策略
1. **文本摘要**：提取关键信息以文本形式发送
2. **关键数据**：只分享最重要的结果和结论
3. **文件位置**：告知用户文件位置，由用户自行查看
4. **明确指令**：只有用户明确要求时才发送文件

## 🚫 **禁止的文件发送行为**

### ❌ 主动发送完整文件
```text
# 错误做法
<qqfile>workspace_analysis.txt</qqfile>
<qqfile>TOOLS.md</qqfile>
<qqfile>code_statistics_workflow.md</qqfile>
```

### ❌ 未经请求的文件附件
```text
# 错误做法
"这是完整的报告文件" <qqfile>report.pdf</qqfile>
```

### ❌ 大文件传输
```text
# 错误做法
"这里有100MB的数据文件" <qqfile>data.zip</qqfile>
```

## 🚨 **例外情况处理**

### 情况1：用户明确要求发送文件
```text

# 正确做法：发送文件

```

### 情况2：文件确实很小且必要
```text
# 非常小的配置文件或脚本
# 正确做法：在获得确认后发送
"需要发送这个配置文件吗？[文件大小: 2KB]"
```

### 情况3：紧急情况需要共享
```text
# 紧急错误日志或配置
# 正确做法：先询问
"发现错误配置，是否需要查看完整的配置文件？"
```
