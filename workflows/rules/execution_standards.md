# 📜 执行标准和质量要求

## 🎯 **核心原则**
**所有工作流执行都应达到高质量标准：正确性、可靠性、一致性**

## ✅ **质量标准**

### 1. **正确性要求**
- **功能正确**：脚本必须正确实现预定功能
- **数据准确**：统计结果必须精确无误
- **逻辑完整**：处理所有边界情况和异常

### 2. **可靠性要求**
- **稳定运行**：脚本应在各种环境下稳定运行
- **错误处理**：完善的错误检测和处理机制
- **资源管理**：合理管理内存、CPU、文件句柄等资源

### 3. **一致性要求**
- **接口一致**：相似的脚本应有相似的接口
- **输出一致**：相同输入应产生相同输出
- **行为一致**：在不同时间、环境下行为一致

## 📋 **脚本开发标准**

### 1. **代码结构**
```python
# 必须包含：
# 1. 明确的文件头注释
# 2. 完整的模块导入
# 3. 清晰的类/函数定义
# 4. 完整的错误处理
# 5. 适当的日志记录
```

### 2. **文档要求**
```python
"""
必须包含：
1. 脚本用途说明
2. 参数说明
3. 使用方法示例
4. 输出格式说明
5. 错误处理说明
"""
```

### 3. **错误处理**
```python
# 必须处理：
try:
    # 主逻辑
except FileNotFoundError:
    logger.error("文件不存在")
except PermissionError:
    logger.error("权限不足")
except Exception as e:
    logger.error(f"未知错误: {e}")
```

### 4. **日志记录**
```python
# 必须记录：
logger.info("开始执行任务")
logger.debug("详细调试信息")
logger.warning("警告信息")
logger.error("错误信息")
logger.info("任务完成")
```

```python

# 1. 使用哪个脚本或创建什么脚本？

# 4. 错误处理如何设计？
```


