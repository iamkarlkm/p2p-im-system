# 分布式数据库学习笔记

## 概述
分布式数据库是即时通讯(IM)系统的核心组成部分，主要用于存储用户信息、消息历史、好友关系、群组数据等。

## 核心组件

### 1. 消息存储
- **消息持久化**：确保消息不丢失
- **消息历史**：支持查看历史消息
- **未读计数**：实时更新未读消息数

### 2. 关系数据存储
- **好友关系**：用户好友列表
- **群组数据**：群成员、群信息
- **黑名单/收藏**：用户个性化数据

### 3. 用户数据
- **用户信息**：Profile、头像、状态
- **登录凭证**：Token、会话管理

## 技术选型

### 常用数据库
1. **MySQL** - 关系型数据库，适合结构化数据
2. **MongoDB** - 文档型数据库，适合消息存储
3. **Redis** - 内存数据库，适合缓存和实时数据
4. **Cassandra** - 分布式数据库，适合高写入场景

### 分库分表策略
- **按用户ID分片**：消息按发送者/接收者ID路由
- **按时间分表**：历史消息按月份/年份归档
- **读写分离**：主从复制提高读取性能

## IM系统数据库架构

```
┌─────────────┐     ┌─────────────┐
│   应用服务   │────▶│   Redis     │ (缓存层)
└─────────────┘     └─────────────┘
       │                   │
       ▼                   ▼
┌─────────────┐     ┌─────────────┐
│  MySQL主库   │◀───▶│  MySQL从库   │
│ (写入/更新)  │     │ (读取查询)   │
└─────────────┘     └─────────────┘
       │
       ▼
┌─────────────┐
│ MongoDB     │ (消息历史)
│ (归档存储)  │
└─────────────┘
```

## 高并发优化

1. **缓存策略**
   - 热点数据放Redis
   - 好友列表、群列表缓存
   - 未读数计数缓存

2. **写入优化**
   - 批量写入
   - 异步写入
   - 消息合并

3. **读取优化**
   - 读写分离
   - 分页优化
   - 索引优化
