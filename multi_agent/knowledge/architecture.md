# 即时通讯系统架构设计文档

## 1. 系统概述

### 1.1 项目目标
基于 Tauri + Flutter + Spring Boot/Cloud + Netty + 分布式数据库的即时通讯系统

### 1.2 技术栈
- **桌面端**: Tauri (Rust + Web)
- **移动端**: Flutter
- **后端**: Spring Boot + Spring Cloud + Netty
- **数据库**: 分布式 MySQL + Redis
- **消息队列**: Kafka
- **缓存**: Redis Cluster
- **负载均衡**: Nginx

## 2. 模块设计

### 2.1 用户模块 (User Service)
- 用户注册/登录
- 用户信息管理
- 好友关系管理
- 黑名单管理

### 2.2 消息模块 (Message Service)
- 单聊消息
- 群聊消息
- 消息存储与同步
- 消息已读/未读状态
- 消息撤回/删除

### 2.3 群组模块 (Group Service)
- 创建群组
- 群成员管理
- 群权限管理
- 群消息分发

### 2.4 关系模块 (Relation Service)
- 好友列表
- 关注/粉丝
- 分组管理

### 2.5 网关模块 (Gateway)
- HTTP API网关
- WebSocket网关 (Netty)
- 鉴权与授权
- 限流与熔断

## 3. 通信协议

### 3.1 WebSocket消息格式
```json
{
  "type": "message|presence|ack",
  "from": "userId",
  "to": "userId|groupId",
  "content": "message content",
  "timestamp": 1234567890,
  "msgId": "unique message id"
}
```

### 3.2 REST API设计
- `POST /api/auth/login` - 登录
- `POST /api/auth/register` - 注册
- `GET /api/users/{id}` - 获取用户信息
- `POST /api/messages` - 发送消息
- `GET /api/messages/{chatId}` - 获取消息历史

## 4. 数据库设计

### 4.1 MySQL表结构
- `users` - 用户表
- `friends` - 好友关系表
- `messages` - 消息表
- `groups` - 群组表
- `group_members` - 群成员表

### 4.2 Redis数据结构
- `user:session:{userId}` - 用户会话
- `user:online` - 在线用户集合
- `message:cache:{chatId}` - 消息缓存

## 5. 安全性设计

### 5.1 认证
- JWT Token认证
- 短期access token + 长期refresh token

### 5.2 加密
- 消息内容端到端加密
- 传输层TLS加密

### 5.3 权限控制
- 基于角色的访问控制 (RBAC)
- 群组权限细分

## 6. 性能优化

### 6.1 消息处理
- 消息异步写入

### 6.2 缓存策略
- 多级缓存架构
- 热点数据预加载

### 6.3 水平扩展
- 微服务架构
