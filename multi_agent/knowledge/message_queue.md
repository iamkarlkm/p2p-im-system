# 消息队列学习笔记

## 概述
消息队列(MQ)是IM系统中不可或缺的组件，用于解耦、削峰、异步通信。在IM系统中，消息队列主要用于消息推送、离线消息处理、消息通知等场景。

## 常用消息队列

### 1. Kafka
- **特点**：高吞吐、低延迟、分布式
- **适用场景**：日志收集、实时流处理、大数据管道
- **优点**：性能高、生态丰富、持久化好
- **缺点**：顺序保证复杂、延迟稍高

### 2. RabbitMQ
- **特点**：功能完善、社区活跃、支持多种协议
- **适用场景**：企业级应用、复杂路由
- **优点**：功能丰富、易于使用、文档完善
- **缺点**：吞吐量相对较低

### 3. RocketMQ
- **特点**：阿里巴巴开源、金融级可靠性
- **适用场景**：电商、金融、IM系统
- **优点**：高可靠、事务消息、支持延迟消息
- **缺点**：生态相对较小

## 消息队列在IM中的应用

### 1. 消息推送
- 实时消息通过MQ推送给在线用户
- 离线消息存储后等待用户上线拉取

### 2. 消息路由
- 群消息广播
- 跨机房消息同步

### 3. 消息确认
- 消息投递确认
- 消息消费ACK

### 4. 延迟消息
- 消息撤回(延迟N秒后撤回)
- 定时提醒
- 会话过期

## 技术选型建议

### IM系统推荐
- **RocketMQ**：支持延迟消息、事务消息，适合IM场景
- **Kafka**：高吞吐，适合消息量大的场景

### 选型因素
1. **消息可靠性**：是否允许消息丢失
2. **吞吐量**：每秒消息量
3. **延迟要求**：毫秒级还是秒级
4. **功能需求**：延迟消息、事务消息、顺序消息
