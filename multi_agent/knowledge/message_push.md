# 消息推送学习笔记

## 概述
消息推送是IM系统中非常重要的功能，用于将消息实时送达用户设备。即使应用在后台或关闭状态，也需要通过推送通道发送通知。

## 推送通道

### 1. 苹果APNs
- **全称**：Apple Push Notification Service
- **特点**：稳定、可靠、无需用户授权
- **适用**：iOS设备
- **原理**：应用 -> APNs -> 设备

### 2. 安卓厂商通道
- **华为Push**：华为设备
- **小米Push**：小米设备
- **OPPO Push**：OPPO设备
- **vivo Push**：vivo设备
- **厂商通道优势**：系统级推送、后台保活

### 3. 第三方推送
- **极光推送**：支持多平台
- **个推**：国内领先
- **友盟推送**：阿里系
- **Firebase Cloud Messaging**：海外

## 推送类型

### 1. 系统通知
- 离线消息提醒
- 新好友请求
- 群消息@提醒

### 2. 自定义消息
- 业务数据透传
- 应用内消息

### 3. 静默推送
- 不弹出通知
- 后台更新数据
- 适用：iOS后台刷新

## 推送优化

### 1. 消息合并
- 多个消息合并为一条
- 减少推送次数

### 2. 离线消息
- 用户上线后拉取
- 未读数同步

### 3. 推送策略
- 免打扰时段
- 消息聚合
- 优先级设置

### 4. 推送成功率
- 设备Token维护
- 通道备用
- 重试机制

## 推送与IM整合

### 在线消息
- 长连接实时送达
- 不需要推送

### 离线消息
- 推送通知提醒
- 用户点击后拉取

### 消息确认机制
- 推送回执
- 消息送达确认
